# CSP竞品情报系统 - 产品逻辑梳理

## 一、产品定位

**CSP (Competitive Intelligence System)** 是一款专为奢侈品牌设计的竞品情报采集与分析平台，帮助品牌通过BA（美容顾问）在企微小程序端实时采集竞品销售、促销、柜台等信息，经审核后汇总分析，为管理层决策提供数据支持。

## 二、目标用户

| 用户角色 | 使用场景 | 核心需求 |
|---------|---------|---------|
| BA (美容顾问) | 门店日常填报 | 快速上报、数据便捷输入、历史查看 |
| 店长 | 审核管理 | 数据审核、质量监控、异常预警 |
| 品牌经理 | 数据分析 | 竞品对比、趋势分析、报表导出 |
| 总部管理员 | 系统管理 | 品牌配置、权限管理、基础信息维护 |

## 四、核心功能模块

### 4.1 前台 - BA端（企微小程序）

| 功能模块 | 功能说明 |
|---------|---------|
| **身份识别** | 自动获取BA所属品牌、柜台信息 |
| **日报上报** | 每日竞品业绩数据填报（含大单、异常状态） |
| **周报上报** | 每周竞品销售汇总分析 |
| **月报上报** | 每月竞品业绩对标分析 |
| **促销上报** | 竞品促销活动信息采集 |
| **柜台维护** | 竞品柜台状态更新（新开/装修/撤柜） |
| **历史查看** | 提交记录、审核状态查询 |
| **智能提醒** | 红点提醒、待办通知、审核反馈 |
| **语音输入** | AI语音识别，自动转换为文字 |
| **文件上传** | 支持图片/Excel批量导入 |

### 4.2 后台 - 管理端（Web系统）

| 功能模块 | 功能说明 |
|---------|---------|
| **数据审核** | 三级审核：BA提交→店长审核→总部确认 |
| **质量监控** | 异常数据识别、未上报柜台追踪、预警提醒 |
| **数据报表** | 日/周/月竞品报告，支持下载导出 |
| **基础信息** | 品牌、柜台、商场信息维护 |
| **规则配置** | 数据修改规则、上报周期规则 |
| **角色权限** | 集团级/品牌级权限分配 |
| **BI看板** | Quick BI可视化数据分析 |

## 五、业务流程

### 5.1 数据上报流程

```
BA(门店)
   │
   ▼ [企微小程序-日报/周报/月报/促销上报]
店长
   │
   ▼ [审核-数据校验-异常标记]
总部
   │
   ▼ [数据整合-分析-报表输出]
管理层决策
```

### 5.2 质量监控流程

```
数据采集 ──► 规则校验 ──► 异常识别 ──► 预警通知
                    │
                    ▼
              未上报柜台统计
              数据异常检测
              趋势偏离预警
```

## 六、数据模型

### 6.1 核心实体

| 实体 | 字段 | 说明 |
|-----|------|------|
| **品牌** | brand_id, brand_name, brand_alias | 品牌基本信息 |
| **商场** | mall_id, mall_name, city, region | 商场地理位置 |
| **柜台** | counter_id, brand_id, mall_id, status | 竞品柜台状态 |
| **BA** | ba_id, name, brand_id, counter_id | 美容顾问信息 |
| **日报数据** | report_id, ba_id, date, sales, competitor_data | 日常上报 |
| **促销信息** | promo_id, brand_id, mall_id, promo_type, period | 促销活动 |
| **审核记录** | audit_id, report_id, status, auditor, comment | 审核流水 |

### 6.2 状态枚举

| 状态 | 值 | 说明 |
|-----|-----|-----|
| 上报状态 | pending, approved, rejected | 待审核/已通过/已驳回 |
| 柜台状态 | normal, new, renovation, closed | 正常/新开/装修/撤柜 |
| 预警级别 | info, warning, critical | 提示/警告/严重 |

## 七、技术架构

### 7.1 技术栈

| 层级 | 技术选型 | 用途 |
|-----|---------|-----|
| **前端** | 微信小程序 + Web后台 | 用户交互 |
| **后端** | Java/Python 微服务 | 业务逻辑 |
| **数据** | Hologres + DataWorks | 数据存储与处理 |
| **报表** | Quick BI | 可视化分析 |
| **AI服务** | Model Service (语音/OCR) | 智能输入 |
| **消息** | 企微API | 通知推送 |

### 7.2 API设计

| 接口 | 方式 | 说明 |
|-----|------|------|
| `/api/report/submit` | POST | 提交上报数据 |
| `/api/report/audit` | POST | 审核操作 |
| `/api/report/list` | GET | 查询上报列表 |
| `/api/dashboard/summary` | GET | 数据看板汇总 |
| `/api/quality/check` | GET | 质量检测结果 |
| `/api/alerts/list` | GET | 预警列表 |

### 7.3 安全机制

- 企微身份认证（OAuth2.0）
- API网关鉴权
- 数据加密传输（HTTPS）
- 敏感数据脱敏

## 八、关键业务流程决策

| 决策点 | 方案 | 原因 |
|-------|------|-----|
| **部署位置** | 阿里云LANDING ZONE | 内网访问、安全合规 |
| **语音识别** | 百炼Model Service | TOKEN计费、成本可控 |
| **审核方式** | 企微小程序内完成 | 减少邮件通知、提升效率 |
| **数据迁移** | 5月底切换、7月1日完成 | 平滑过渡、保障015不中断 |
| **报表工具** | Quick BI | 集成阿里生态、性能稳定 |
