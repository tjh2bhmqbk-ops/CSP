# CSP竞品情报系统 - 首页内容规划（最终版）

## 一、首页定位

首页是系统的"指挥中心"，让用户登录后第一时间掌握核心数据、待办事项和异常预警，快速进入高频操作。

---

## 二、用户角色确认（严格基于PDF功能地图 PAGE 11）

| 角色 | 入口 | 职责 |
|-----|------|------|
| **导购BA** | 企微小程序 | 数据录入：日报上报、月报上报、历史数据查看/修改 |
| **店长/督导** | 企微小程序 | 审核管理：审批上报数据（日报/周报/月报/促销/柜台维护）、质量监控 |
| **总部管理员** | Web后台 | 系统管理：品牌管理、区域管理、报表分析、异常预警 |

## 三、首页模块设计（严格遵循PDF）

### 3.1 导购端BA首页（企微小程序）

**设计要点（严格基于PDF PAGE 11-16）：**
- 首页显示**门店**基础信息，而非个人身份
- 以**门店为单位**上报，任何被指定的BA登录后都可见门店填报提醒
- 异常数据提醒、提交状态提醒、审批反馈提醒

**BA首页核心要素（严格按PDF）：**
| 模块 | 内容 |
|-----|------|
| 门店信息 | 品牌、门店名称、BA姓名 |
| 今日待办 | 日报/月报未提交提醒 |
| 数据上报 | **仅日报、月报两个入口**（无周报、无促销采集） |
| 上报进度 | 本月累计上报天数/应上报天数 |
| 待处理事项 | 审核状态反馈、需修改项 |
| 历史记录 | 日/月历史数据查询，可修改后重新提交 |

**❌ 已确认不包含的功能：**
- 无周报上报（PDF PAGE 11功能地图只有日/月）
- 无AI语音识别
- 无临时支援身份显示
- 无竞品促销活动采集入口
