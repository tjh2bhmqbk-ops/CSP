## 二、修正计划

### 2.1 第一阶段：设计风格校准（优先级：高）

#### 1. 统一设计语言
- 深色渐变头部（品牌蓝）
- 简洁功能图标
- 卡片模块化布局
- 统一的提醒标识（红点）

#### 2. 导航结构优化
- 底部Tab导航（首页/上报/我的）
- 功能模块卡片入口

#### 3. 配色方案（待确认）
- 主色调：品牌蓝
- 辅助色：功能灰
- 警示色：红/黄/绿

### 2.2 第二阶段：BA端功能补全（优先级：高）

| 页面 | 功能 | 说明 |
|------|------|------|
| BA首页 | 每日上报入口 | 每日竞品数据录入 |
| BA首页 | 每周上报入口 | 每周竞品数据汇总 |
| BA首页 | 每月上报入口 | 每月竞品数据对标 |
| BA首页 | 竞品柜台信息维护入口 | 竞品柜台状态上报 |
| 日数据查看修改 | 历史日报查询 | 查看、修改已提交数据 |
| 周数据查看修改 | 历史周报查询 | 查看、修改已提交数据 |
| 月数据查看修改 | 历史月报查询 | 查看、修改已提交数据 |
| 提醒机制 | 提交/审核状态提醒 | 红点提醒标识 |

### 2.3 第三阶段：店长端功能补全（优先级：高）

| 页面 | 功能 | 说明 |
|------|------|------|
| 店长首页 | 竞品数据待审批提醒 | 消息推送入口 |
| 店长首页 | 未上报柜台列表 | 查看未上报柜台 |
| 店长首页 | 数据异常柜台 | 查看异常数据 |

## 三、Demo文件规划

```
/demo/
├── index.html              # Demo入口（角色选择）
│
├── ba/                     # 门店BA端（企微小程序风格）
│   ├── index.html         # BA首页
│   ├── daily-report.html  # 每日上报
│   ├── weekly-report.html # 每周上报
│   ├── monthly-report.html# 每月上报
│   ├── counter-info.html  # 竞品柜台信息维护
│   ├── daily-history.html # 日数据查看修改
│   ├── weekly-history.html# 周数据查看修改
│   └── monthly-history.html# 月数据查看修改
│
├── manager/               # 店长/督导端（企微小程序风格）
│   ├── index.html         # 店长首页
│   ├── review-center.html # 审核中心
│   ├── counter-review.html# 竞品柜台维护审批
│   ├── missing-list.html  # 未上报柜台
│   └── abnormal-list.html# 数据异常柜台
│
└── admin/                 # 管理后台（Web系统）
    ├── index.html         # 管理首页
    ├── alert-center.html  # 异常预警中心
    ├── reports.html       # 数据报表
    ├── review-admin.html  # 修改审批
    ├── brand.html         # 品牌管理
    ├── competitor.html    # 竞品关系管理
    ├── region.html        # 区域/城市管理
    └── counter.html       # 柜台管理
```
